7. Como actualizar la base real desde un CSV

Si tenes un CSV nuevo:
1. Subirlo desde la pestaña Files a PythonAnywhere
2. Anotar su ruta completa

Ejemplo:
   /home/cprargentina/uploads/donors_nuevo.csv

En la Bash console:

   workon cpra-venv
   cd ~/cPRArgentina
   python load_donors.py --csv "/home/cprargentina/uploads/donors_nuevo.csv" --mode append --db cpra.db

Eso hace:
- agrega solo donor_id nuevos
- ignora donor_id ya existentes

Si alguna vez queres reconstruir toda la base desde cero:

   python load_donors.py --csv "/home/cprargentina/uploads/donors_nuevo.csv" --mode rebuild --db cpra.db

Eso hace:
- backup de la base previa
- borra la base anterior
- recrea cpra.db desde el CSV

Despues de cualquier carga:
1. Ir a Web
2. Click en Reload
3. Probar /health

10. Problemas tipicos

- Problema:
  /health muestra cpra_demo.db
  Solucion:
  revisar que asgi.py exista y que siga fijando CPRA_DB=cpra.db

- Problema:
  total_donors no cambio despues de subir CSV
  Solucion:
  correr load_donors.py y despues hacer Reload en Web

- Problema:
  antigeno valido no reconocido
  Solucion:
  revisar data/hla_validation.csv y confirmar que este incluido

- Problema:
  codigo actualizado pero la web sigue igual
  Solucion:
  hacer git pull en PythonAnywhere y despues Reload


11. Seguridad basica

- No pegar API tokens en chats o mails
- Si un token se expone, regenerarlo
- No subir la base real a GitHub
- No subir CSVs reales al repo


12. Regla practica para no romper nada

Orden recomendado cuando haya cambios:
1. actualizar codigo en GitHub
2. git pull en PythonAnywhere
3. si hay cambios de datos, cargar CSV a cpra.db
4. Reload en la pestaña Web
5. probar /health
6. probar la home
